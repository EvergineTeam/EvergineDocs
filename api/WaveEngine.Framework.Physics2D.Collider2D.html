    GroupIndex

    Gets or sets the group index of the collision

    Remarks

    The GroupIndex flag of a collider can be used to override the category and mask settings. As the name implies it can be useful to group together fixtures that should either always collide, or never collide. The groupIndex is used as a signed integer instead of a bitflag. Here's how it works - read it slowly because it can be a bit confusing at first. When checking two fixtures to see if they should collide:

    • If either fixture has a groupIndex of zero, use the category/mask rules as above
    • If both groupIndex values are non-zero but different, use the category/mask rules as above
    • If both groupIndex values are the same and positive, collide
    • If both groupIndex values are the same and negative, don't collide
